# User Stories

As *User Stories* foram elaboradas com base nas entrevistas realizadas aos *stakeholders*, cujas transcrições se encontram no Anexo B. Seguem o formato padrão: _"Como [papel], quero [funcionalidade], para que [benefício]."_ A cada *User Story* foi atribuído um identificador único e uma prioridade segundo o método MoSCoW (*Must Have*, *Should Have*, *Could Have*, *Won't Have*).

## Diretor / Gestor

| ID    | User Story | Prioridade |
|-------|------------|------------|
| US-01 | Como diretor, quero consultar a disponibilidade em tempo real dos alojamentos e a taxa de ocupação atual, para ter uma visão operacional do hotel. | Must Have |
| US-02 | Como diretor, quero consultar indicadores de faturação e pagamentos pendentes filtráveis por período, para acompanhar o desempenho financeiro do hotel. | Must Have |
| US-03 | Como diretor, quero gerir as contas e perfis de acesso dos colaboradores — criando, editando e desativando colaboradores —, para garantir que cada um acede apenas às funcionalidades necessárias ao exercício das suas funções. | Must Have |
| US-04 | Como diretor, quero gerar relatórios operacionais e financeiros por período, com filtros, agrupamentos e exportação, para apoiar a tomada de decisão e o planeamento do negócio. | Must Have |
| US-05 | Como diretor, quero consultar o histórico completo de reservas, estadias, pagamentos e serviços associados, para ter uma visão financeira e operacional do hotel. | Must Have |
| US-06 | Como diretor, quero gerir os tipos de alojamento, as respetivas tarifas diárias e os tipos de serviços extra disponíveis, para garantir que reservas, check-in, check-out e relatórios usam valores configurados e rastreáveis. | Must Have |
| US-07 | Como diretor, quero consultar o registo de auditoria das operações críticas do sistema, para saber quem realizou ações como reservas, check-in, check-out, pagamentos, limpeza, serviços extra e intervenções clínicas. | Should Have |

## Funcionário de Receção

| ID    | User Story | Prioridade |
|-------|------------|------------|
| US-08 | Como funcionário de receção, quero criar e gerir reservas com controlo automático de disponibilidade, para evitar situações de overbooking. | Must Have |
| US-09 | Como funcionário de receção, quero registar o check-in e check-out de cada animal respeitando a sequência obrigatória e impedindo estadias ativas duplicadas para o mesmo animal, para garantir o controlo correto das estadias em curso. | Must Have |
| US-10 | Como funcionário de receção, quero consultar o histórico de reservas, estadias, pagamentos, notas e serviços de cada animal, para prestar um serviço mais personalizado e informado. | Should Have |
| US-11 | Como funcionário de receção, quero registar e consultar os dados dos tutores e dos seus animais, para ter toda a informação disponível no momento do atendimento. | Must Have |
| US-12 | Como funcionário de receção, quero registar o pagamento da estadia no momento do check-in, com base na duração reservada e no tipo de alojamento, para que o valor base fique liquidado à entrada. | Must Have |
| US-13 | Como funcionário de receção, quero registar o check-out de um animal e cobrar nesse momento os serviços extra, intervenções veterinárias e dias adicionais acumulados durante a estadia, para que a faturação complementar fique liquidada à saída. | Must Have |
| US-14 | Como funcionário de receção, quero consultar a disponibilidade das boxes em tempo real, considerando reservas sobrepostas, estadias ativas e estado de limpeza, para responder de imediato a pedidos de reserva sem criar conflitos. | Must Have |
| US-15 | Como funcionário de receção, quero registar serviços opcionais — como banhos e passeios — durante a estadia, para que fiquem associados ao acompanhamento do animal e possam ser incluídos na faturação final quando tiverem custo. | Could Have |

## Cuidador de Animais

| ID    | User Story | Prioridade |
|-------|------------|------------|
| US-16 | Como cuidador, quero consultar o plano de cuidados ativo de cada animal em estadia, incluindo tarefas recorrentes, instruções específicas e prioridade atual, para garantir que todos recebem os cuidados corretos. | Must Have |
| US-17 | Como cuidador, quero registar os cuidados prestados a cada animal durante uma estadia ativa, com data, hora, autor e descrição/observações, para garantir rastreabilidade e continuidade entre turnos. | Must Have |
| US-18 | Como colaborador autorizado, quero registar alterações ao estado de saúde de um animal durante a estadia, com severidade e descrição, para que a informação esteja disponível para o médico veterinário, para a receção e para ajustar a prioridade do plano de cuidados. | Must Have |
| US-19 | Como cuidador ou funcionário de receção, quero adicionar notas operacionais a uma reserva, para transmitir informação relevante sobre o animal, como instruções do tutor ou observações do turno, aos colegas que acompanham a estadia. | Must Have |
| US-20 | Como cuidador, quero registar um serviço extra realizado durante uma estadia ativa e associar o respetivo custo à reserva em curso, para que seja incluído na faturação do check-out. | Must Have |
| US-21 | Como cuidador, quero consultar e confirmar a execução dos serviços de banho e passeio agendados para o meu turno, para garantir que nenhum serviço é omitido e que o registo de conclusão fica disponível para faturação. | Could Have |

## Responsável pela Limpeza

| ID    | User Story | Prioridade |
|-------|------------|------------|
| US-22 | Como responsável pela limpeza, quero consultar a lista de alojamentos que necessitam de limpeza após um check-out, para organizar o meu trabalho de forma eficiente. | Must Have |
| US-23 | Como responsável pela limpeza, quero marcar um alojamento como limpo no sistema, para que fique disponível para novas reservas. | Must Have |

## Médico Veterinário

| ID    | User Story | Prioridade |
|-------|------------|------------|
| US-24 | Como médico veterinário, quero consultar o historial clínico e operacional consolidado de cada animal, filtrável por animal, cliente, estadia, intervalo de datas e tipo de evento, para fundamentar decisões de tratamento com informação completa. | Must Have |
| US-25 | Como médico veterinário, quero registar intervenções, prescrições e o custo associado durante uma estadia ativa, identificando o responsável clínico, para que a informação esteja disponível para todos os cuidadores e o custo seja incluído na faturação da estadia. | Must Have |
| US-26 | Como médico veterinário, quero consultar uma lista de animais com alterações recentes no estado de saúde, para poder intervir atempadamente. | Should Have |
