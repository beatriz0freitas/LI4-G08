# UC-08 - Processar Pagamento

- Use Case: Processar Pagamento
- Descricao: Caso de uso incluido para registar pagamentos associados ao check-in ou ao check-out.
- Cenario: O funcionario apresenta ao tutor o valor a liquidar e regista o pagamento ou a pendencia.
- Pre-condicao: Existe um valor apurado no caso de uso que invocou o pagamento.
- Pos-condicao: Pagamento registado com valor, metodo, estado e associacao a estadia.
- Fluxo Normal:
  1. Sistema apresenta o valor a cobrar.
  2. Funcionario indica o metodo de pagamento.
  3. Funcionario confirma a liquidacao.
  4. Sistema regista o pagamento como liquidado.
- Fluxo Alternativo:
  - [Pagamento fica pendente] (passo 3)
    1. Funcionario indica que o pagamento nao ficou liquidado.
    2. Sistema regista o pagamento como pendente.
- Origem: RF-10; RD-04
